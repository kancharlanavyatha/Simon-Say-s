# Simon Game

A web-based implementation of the classic Simon memory game. This project challenges players to remember and repeat increasingly complex sequences of colors and sounds.

---

## Project Overview

The Simon Game is a logic-based memory challenge. The game generates a random sequence of flashes. The player must reproduce that sequence by clicking the corresponding buttons. With every successful round, the sequence grows longer, testing the limits of the player's short-term memory.



### Core Features

* **Dynamic Sequencing:** The game randomly generates a new color for the sequence at each level.
* **Progress Tracking:** An interactive header displays the current level and provides feedback on game status.
* **Visual Feedback:** Custom CSS animations provide immediate visual confirmation when buttons are triggered by the game or clicked by the user.
* **State Management:** The logic handles game starts, sequence verification, and game-over states with the ability to restart instantly.

---

## How to Play

1.  **Start:** Press any key on your keyboard to begin the game.
2.  **Observe:** Watch the sequence generated by the game. Each level adds one new color to the existing pattern.
3.  **Repeat:** Click the buttons in the exact same order as shown.
4.  **Advance:** If you match the pattern correctly, the game will wait one second and then move to the next level.
5.  **Game Over:** If you click the wrong button, the header will update to "Game Over." You can then press any key to reset and try again.

## Technical Details

The application is built using a clean, vanilla stack to ensure performance and compatibility.

### Technologies Used

| Component | Description |
| :--- | :--- |
| **HTML5** | Defines the structure of the game board and interactive elements. |
| **CSS3** | Handles the styling, layout using Flexbox, and button animations. |
| **JavaScript** | Manages the game logic, event listeners, and sequence validation. |

### Logic Flow

The game operates on a state-based logic system:
* **Pattern Generation:** Uses `Math.random()` to select from an array of colors.
* **User Input:** Captures clicks and stores them in a temporary array for comparison.
* **Validation:** Compares the user's current index against the game's stored pattern in real-time.
